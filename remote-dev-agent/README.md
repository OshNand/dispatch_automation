# Remote Autonomous Development Agent

A LOCAL-FIRST "Remote Autonomous Development Agent with Monitoring & Safety Control".

This system allows you to send master development prompts via Telegram to your local workstation. The agent will translate your prompt into atomic execution sessions, plan the steps using a local LLM (Ollama), and execute them securely while monitoring system resources to prevent overload.

## Architecture

- **bot/**: Telegram bot interface. Handles authentication, receives master prompts, provides system status, and handles interactive inline-keyboard approvals.
- **agent/**: Execution engine. Iterates over the steps generated by the planner and interacts with the workspace via tools.
- **reasoning/**: Translates master prompts into structured sessions, and plans step-by-step actions for each session. Uses Ollama API.
- **monitor/**: System monitoring and safety controller. Pauses operations if CPU, GPU, or Temperature limits are exceeded.
- **tools/**: CLI tools for safe file and command execution within the workspace.
- **config/**: System configuration (settings.py).

## Setup Instructions

1. **Install Prerequisites**:
   - Python 3.10+
   - Ollama (installed locally)

2. **Pull LLM**:
   ```bash
   ollama pull gemma
   ```

3. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Configuration**:
   You can either edit `config/settings.py` directly, or set the following environment variables:
   - `TELEGRAM_BOT_TOKEN`: Your Telegram Bot Token (from @BotFather).
   - `ALLOWED_USER_ID`: Your numeric Telegram User ID (to prevent others from commanding your PC).
   - *Optional:* `OLLAMA_MODEL` (default: gemma), `OLLAMA_BASE_URL` (default: http://localhost:11434/api).

5. **Run the Agent**:
   ```bash
   python main.py
   ```

## Usage

1. Open Telegram and start a chat with your bot.
2. Send `/start` to verify connection.
3. Send a **Master Prompt** describing what you want to build or modify.
4. The bot will analyze the prompt and present the first planned session.
5. Click **YES** to execute, **SKIP** to skip, or **STOP** to abort.
6. Review the session report after execution.
7. Repeat until all sessions are complete.
8. Send `/status` anytime to check your PC's telemetry (CPU, RAM, GPU, Temp).

## Safety & Security

- **User Validation**: Only responds to `ALLOWED_USER_ID`.
- **Sandbox Context**: File operations are bounded to the `workspace/` directory within this project.
- **Thermal & Load Throttling**: Auto-pauses if system hits 95% CPU/GPU or 85°C.
